# Word Reciter 设计文档

> 版本：v1.0  
> 日期：2026-07-02  
> 状态：已评审，待实现计划

## 1. 项目概述

一款小型桌面端单词背诵辅助软件，帮助用户快速将词库中的单词按熟悉度分类，并通过多种题型复习半熟词和生词。

### 1.1 目标用户
- 需要背诵大量英语单词的学习者。
- 希望快速筛选“已掌握 / 待复习”单词的用户。

### 1.2 核心目标
- 快速分类：用户能高效标记每个单词为熟词、半熟词或生词。
- 针对性复习：背诵模式只出现用户未掌握的单词。
- 多题型巩固：通过选择题、填空题、回想题多种形式加深记忆。
- 离线可用：首次拉取释义后，核心功能无需联网。

## 2. 技术架构

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 桌面端框架 | Tauri v2 | Rust 后端 + Web 前端，产物体积小，支持 Windows / macOS / Linux。 |
| 前端 | Vue 3 + TypeScript + Vite | 组件化开发，Vue Router 切换模式。 |
| 后端 | Rust + Tauri Commands | 本地文件与数据库操作。 |
| 本地存储 | SQLite（tauri-plugin-sql） | 保存单词、用户标记、学习记录。 |
| 中文释义来源 | DeepL API 预生成 | 通过 `scripts/generate_chinese_meanings.py` 生成 `references/unique_words_with_chinese.txt`，应用直接从本地文件读取。 |
| 词库文件 | `references/unique_words.txt` / `references/unique_words_with_chinese.txt` | 默认朗文 9000 词；支持从 `.txt` 导入更多词库。 |

## 3. 数据模型

### 3.1 `words` 表
存储单词基础信息和缓存的释义。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PRIMARY KEY | 自增主键。 |
| `word` | TEXT UNIQUE NOT NULL | 英文单词或短语，唯一。 |
| `source` | TEXT | 来源词库文件名，如 `unique_words_with_chinese.txt`。 |
| `meaning` | TEXT | 中文释义文本。 |
| `created_at` | TIMESTAMP | 记录创建时间。 |

### 3.2 `word_states` 表
存储用户对每个单词的熟悉度。

| 字段 | 类型 | 说明 |
|------|------|------|
| `word_id` | INTEGER PRIMARY KEY | 关联 `words.id`。 |
| `familiarity` | TEXT CHECK | `unknown` / `half` / `known`。 |
| `marked_at` | TIMESTAMP | 首次标记时间。 |
| `updated_at` | TIMESTAMP | 最后更新时间。 |

### 3.3 `study_logs` 表
存储学习记录，用于统计正确率。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PRIMARY KEY | 自增主键。 |
| `word_id` | INTEGER | 关联 `words.id`。 |
| `quiz_type` | TEXT CHECK | `choice` / `fill` / `recall`。 |
| `result` | TEXT CHECK | `correct` / `wrong` / `skipped`。 |
| `familiarity_after` | TEXT CHECK | 答题后用户标记的状态。 |
| `created_at` | TIMESTAMP | 记录创建时间。 |

## 4. 功能设计

### 4.1 首页 / 词库选择
- 显示默认词库“朗文 9000 词”。
- 显示学习统计：生词数、半熟词数、熟词数、总单词数。
- 提供“开始分类模式”和“开始背诵模式”两个入口。
- 提供“导入新词库”入口（未来扩展）。

### 4.2 分类模式
- 快速展示一个英文单词。
- 用户标记为：生词 / 半熟词 / 熟词。
- 支持键盘快捷键：1=生词，2=半熟词，3=熟词。
- 支持“跳过”按钮，不标记直接进入下一个。
- 顶部显示当前进度（如 `43 / 9,411`）。

### 4.3 背诵模式
- 从 `familiarity != 'known'` 的单词中随机出题。
- 仅使用具备有效中文释义的单词；无释义单词暂时跳过。
- 三种题型按 4:3:3 比例随机混合：
  1. **看英文选中文**：给出英文单词，4 个中文释义选项。
  2. **看中文 + 首尾字母填空**：如 `a__le` → `apple`。
  3. **看英文想中文**：仅显示英文单词，用户自行回想后显示答案并自评对错。
- 每题答完后显示正确答案，并弹出重新标记面板：生词 / 半熟词 / 熟词。
- 标记为熟词的单词从背诵池移除。
- 顶部显示当前题号与本局正确率。

## 5. 状态流转

```
单词初始状态：未标记（分类模式首次出现）
          ↓
    ┌─────┼─────┐
    ↓     ↓     ↓
  生词  半熟词  熟词
    ↓     ↓     ↓
  进入背诵池   不再出现
  （生词 + 半熟词）
          ↓
    背诵模式答题
          ↓
    重新标记熟悉度
          ↓
    熟词 → 移出背诵池
```

## 6. 界面结构

- **首页**：词库选择 + 统计 + 模式入口。
- **分类模式页**：中央大单词 + 三个标记按钮 + 跳过 + 进度。
- **背诵模式页**：题目区 + 作答区 + 提交 + 答案与重新标记面板。
- **公共导航栏**：可随时返回首页。
- **提示信息**：无释义单词跳过、背诵池为空时给出友好提示。

## 7. 题型细节

### 7.1 多选题
- 干扰项从当前背诵池随机取 3 个单词的中文释义。
- 若背诵池释义不足，随机选取其他单词释义作为干扰项。
- 选择后立即显示对错，进入重新标记面板。

### 7.2 填空题
- 显示中文释义 + 英文首尾字母，中间用下划线代替。
- 输入框不区分大小写，忽略首尾空格。
- 允许少量拼写容错（如忽略末尾标点）。
- 提交后显示完整单词，进入重新标记面板。

### 7.3 回想题
- 仅显示英文单词，用户点击“显示答案”。
- 用户自行判断对错，然后进入重新标记面板。

## 8. 错误处理与边界情况

- **无中文释义**：分类模式允许继续标记；背诵模式仅使用有中文释义的单词。
- **缺少中文释义导致背诵池为空**：提示“当前词库缺少中文释义，请检查词库文件”。
- **词库重复导入**：按 `word` 去重，已存在单词保持原有熟悉度标记。
- **背诵池为空**：
  - 全部掌握：提示“恭喜，当前词库已全部掌握”。
  - 缺少释义：提示“当前词库缺少中文释义，请检查词库文件”。
- **极端输入**：填空题空提交计为错误；标记操作同时支持键盘和鼠标。

## 9. 非功能需求

- **平台**：Windows、macOS、Linux。
- **离线使用**：应用核心功能完全无需联网；中文释义在构建/准备阶段通过 DeepL API 预生成。
- **性能**：9,411 单词本地 SQLite 存储，索引保证随机查询效率。
- **隐私**：学习记录仅本地保存；构建阶段向 DeepL API 发送单词文本，运行时不再联网。
- **可扩展性**：支持导入新词库；题型和模式易于新增。
- **开发命令**：`npm run tauri dev` 启动开发环境；`npm run tauri build` 打包。
